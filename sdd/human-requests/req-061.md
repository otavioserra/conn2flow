# req-061: Renomeação para project_tables_config.json e Filtragem de Tabelas no Manifesto

Para desacoplar e diferenciar o arquivo de configurações de sincronização do core do arquivo customizado do projeto, e também para corrigir a poluição de tabelas no manifesto consolidado:

## Mudanças Necessárias

### 1. Renomeação do Arquivo de Configuração do Projeto
- O arquivo de configuração de tabelas customizadas do projeto passa a se chamar **`project_tables_config.json`** (em vez de `tables_config.json`), residindo em `gestor/resources/project_tables_config.json`.
- O arquivo global do core do sistema mantém o nome original **`tables_config.json`** em `gestor/resources/tables_config.json` contendo apenas as 4 tabelas de núcleo (`variaveis`, `usuarios_perfis_modulos`, `usuarios_perfis_modulos_operacoes`, `alvos_ia`).

### 2. Alterações no Compilador de Recursos (`atualizacao-dados-recursos.php`)
- Atualizar a função `coletarConfigsTabelas()` para:
  - Se `--project-path` estiver ativo:
    - Ler as configurações do core de `tables_config.json`.
    - Ler as configurações do projeto de `project_tables_config.json` (usando o rótulo `project:project_tables_config.json`).
  - Se `--project-path` não estiver ativo (modo core):
    - Ler apenas de `tables_config.json`.
- Na função `gerarSchemaMetadata()`, ao gerar o manifesto `project-schema-metadata.json`:
  - Ler `project_tables_config.json`.
  - Obter as chaves declaradas sob a propriedade `"tabelas"`.
  - Filtrar a lista consolidada `$tables` para gravar no `project-schema-metadata.json` **apenas** as chaves declaradas no arquivo do projeto.

### 3. Alterações no Descompilador Cliente (`recuperacao-dados-recursos.php`)
- Atualizar `rdr_coletar_configs()` para:
  - Ler o arquivo global do core `tables_config.json` (se existir) para obter definições de base.
  - Ler o arquivo do projeto `project_tables_config.json` (se existir) para obter/sobrescrever definições do projeto.

### 4. Atualização da Documentação Técnica
- Atualizar `ai-workspace/pt-br/docs/CONN2FLOW-SISTEMA-RECURSOS.md` e `ai-workspace/en/docs/CONN2FLOW-RESOURCES-SYSTEM.md` para documentar que a configuração local do projeto deve residir em `project_tables_config.json`.

## Validação
- Garantir que `php -l` passe.
- Rodar a suíte de testes `composer test`.
- Ajustar os testes em `RecuperacaoDadosRecursosTest.php` para simular o comportamento utilizando o arquivo `project_tables_config.json` nas operações de nível de projeto.
