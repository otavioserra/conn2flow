## 1. Regra de Alimentação Manual no `publisher-index`

### 1.1 Problema
Ao definir a regra de alimentação como "Manual (curadoria por item)" (`rule = "manual"`) no CRUD de `publisher-index`, o widget publicado e o pré-visualizador continuam realizando buscas dinâmicas automáticas de publicações em vez de exibir apenas os itens explicitamente selecionados na curadoria.

### 1.2 Solução no Backend (`publisher-index.widget.php` e AJAX)
- **Leitura da Curadoria**: Nos pontos de renderização (page load e AJAX de paginação/busca em `publisher_index_render_ajax`), extrair `rule` e `selected_items` (array de IDs) do `$schema` de configurações do índice.
- **Lógica de Busca (`publisher_index_widget_buscar_publicacoes`)**:
  - Se `rule === 'manual'`:
    - Se `selected_items` estiver vazio, retornar array vazio `[]` imediatamente.
    - Na query SQL, aplicar o filtro restritivo de ID: `AND p.id IN ('ID1', 'ID2', ...)` (usando `banco_escape_field` em cada ID).
    - Não aplicar ordenação ou limite na query SQL.
    - No PHP, mapear os registros retornados por `page_id` e reordenar a lista resultante para respeitar **exatamente** a ordem contida no array `selected_items`.
    - Caso exista um termo de busca (`busca` não vazio), filtrar os registros em PHP de forma case-insensitive comparando com o título (`titulo`) ou campos customizados do registro.
    - Aplicar a paginação (offset e limit) em PHP sobre a lista ordenada resultante usando `array_slice()`.
- **Lógica de Contagem (`publisher_index_widget_contar_publicacoes`)**:
  - Se `rule === 'manual'`:
    - Se `selected_items` estiver vazio, retornar `0`.
    - Caso exista um termo de busca (`busca`), retornar a quantidade de itens em `selected_items` que casam com o termo (filtrando conforme a lógica de busca). Caso contrário, retornar o `count($selected_items)`.

### 4.1 Pré-visualizador (`html-editor-interface.js`)

  ```javascript
  var bodyHtml = document.body.innerHTML;
  var varRe = /@?\[\[widgets#(.+?)\]\]@?/gi;
  if (varRe.test(bodyHtml)) {
  ```

## 5. Correção de Falha de Teste Unitário JS (Vitest)

O stub de testes simplificado de jQuery (`jquery-stub.js`) não possui essas funções implementadas, resultando no erro `TypeError: $items.children is not a function`.

### 5.2 Solução
- No arquivo de stubs de testes [jquery-stub.js](file:///c:/Users/otavi/OneDrive/Documentos/GIT/conn2flow/tests/Unit/JS/helpers/jquery-stub.js), adicionar os métodos `.children()` e `.not()` na classe interna `JQueryCollection` para simular o comportamento básico do jQuery:
  - **`children(selector)`**: Retorna uma nova `JQueryCollection` contendo os nós filhos directos dos elementos colecionados, opcionalmente filtrados por um seletor.
  - **`not(selector)`**: Retorna uma nova `JQueryCollection` filtrando os elementos da coleção atual que não correspondem ao seletor informado.

## 6. Prefixagem Dinâmica de URL Raiz para Campos de Imagem

### 6.1 Problema
Ao montar os itens para os widgets (`publisher-index` e `publisher-highlights`), os valores dos campos customizados (como caminhos de imagem) salvos na tabela `publisher_pages.fields_values` contêm caminhos relativos (ex: `/arquivos/imagem.png`). Os novos templates que utilizam imagem destaque precisam renderizar o caminho absoluto ou a partir da raiz do site. Para isso, é necessário prependir dinamicamente a URL raiz (`$_GESTOR['url-raiz']`) a esses valores. Como os tipos dos campos não são definidos na tabela `publisher_pages` e sim no campo `fields_schema` da tabela `publisher`, precisamos recuperar dinamicamente se o campo customizado é do tipo `'image'`.

### 6.2 Solução no Backend

- **Consulta ao Schema do Publicador (Cache Estático)**:
  - Recuperar a coluna `fields_schema` da tabela `publisher` filtrando por `id = $publisher_id` e `language = $_GESTOR['linguagem-codigo']`.
  - Mapear a lista de campos (`fields` no JSON) associando cada ID do campo ao seu respectivo `type`.
  - Como otimização para evitar múltiplas consultas ao banco de dados no mesmo request, os tipos de campos por publicador devem ser cacheados em uma variável estática/memória no script PHP.

- **Módulo `publisher-index` (`publisher-index.widget.php`)**:
  - Atualizar a assinatura da função helper de montagem de itens:
    `function publisher_index_widget_montar_itens($rows, $publisher_id = null)`
  - Atualizar todas as chamadas de `publisher_index_widget_montar_itens` para passar o respectivo `$publisher_id` (dentro de `publisher_index_widget_buscar_publicacoes` e `publisher_index_widget_lista_manual`).
  - Em `publisher_index_widget_montar_itens`, caso o `$publisher_id` seja fornecido:
    - Obter a relação de tipos de campo do publicador.
    - Para cada campo customizado em `fields_values` do registro, verificar se seu ID está mapeado como tipo `'image'`.
    - Se for do tipo `'image'` e o valor não estiver vazio, prependir `$_GESTOR['url-raiz']` ao valor retornado.

- **Módulo `publisher-highlights` (`publisher-highlights.widget.php`)**:
  - Na função `publisher_highlights_widget_buscar_publicacoes`, aplicar a mesma lógica de consulta ao `fields_schema` da tabela `publisher`.
  - Ao processar cada campo customizado de `fields_values` do registro, verificar se o ID do campo está mapeado como tipo `'image'`.
  - Se for do tipo `'image'` e o valor não estiver vazio, prependir `$_GESTOR['url-raiz']` ao valor retornado.
