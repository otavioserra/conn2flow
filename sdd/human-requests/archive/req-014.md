# Human Request: Refatoração de Curadoria Manual, Autocomplete Ajax e Reordenação Drag and Drop (req-014)

Esta especificação detalha a substituição do dropdown customizado de curadoria manual por um componente de Autocomplete feito do zero (usando input básico do Fomantic UI, busca dinâmica via AJAX, e labels reordenáveis com drag-and-drop), além de corrigir a inicialização e estilização do editor CodeMirror na aba "Código do Widget".

---

## 1. Escopo das Alterações e Justificativa

### 1.1 Autocomplete de Curadoria do Zero
* **Motivo**: O componente `jquery-custom-dropdown.js` (baseado no dropdown múltiplo do Fomantic UI) apresentou sucessivos problemas de sincronização de estado, concorrência de callbacks de inserção/remoção, e falha na ordenação cronológica dos cliques. 
* **Solução**: Remover completamente o `jquery-custom-dropdown.js`. Em seu lugar, implementar no módulo de curadoria um campo de busca simples (`<input type="text">`), exibindo as sugestões retornadas dinamicamente via AJAX em uma lista flutuante. Ao selecionar uma sugestão, o item é adicionado como uma tag (Fomantic UI Label) abaixo do campo.
* **Ordenação Drag-and-Drop**: As tags adicionadas devem suportar arraste e soltura (Drag-and-Drop) para reordenação manual (usando a biblioteca `Sortable.js`). Toda movimentação ou remoção de tag deve atualizar o array `schema.selected_items` e disparar imediatamente a atualização da pré-visualização ao vivo (`scheduleWidgetPreview`).

### 1.2 Restauração do CodeMirror na Aba do Widget
* **Motivo**: O CodeMirror que exibe o código do widget na aba "Código do Widget" (`#hep-widget-code`) parou de renderizar, voltando a ser um `<textarea>` HTML padrão devido à remoção indevida da função `updateWidgetCodeTab` no lote anterior.
* **Solução**: Restaurar a inicialização automática do CodeMirror com tema escuro (`tomorrow-night-bright`), altura (800px), modo de leitura-apenas (`readOnly: true`), e proteção contra duplicação de instâncias no DOM ao mudar de aba.

### 2.4 Lógica de Controle JavaScript
* **Local**: `gestor/modulos/publisher-highlights/publisher-highlights.js` [MODIFY]
* **Alterações principais**:
  1. **Remover referências antigas**: Eliminar instanciamento, chamadas de métodos, e referências a `manualItemsDropdown` e `ensureManualItemsDropdown()`.
  2. **Inicialização/Hidratação**:
     - Ao carregar a página, se `schema.selected_items` contiver slugs (IDs das publicações) e houver um `publisher_id` selecionado, fazer uma requisição AJAX para buscar os nomes reais dessas publicações.
     - Rota AJAX a utilizar: `publisher-pages-fetch` (já existente no backend). Parâmetros a enviar no payload `data`:
       - `opcao`: `gestor.moduloOpcao`
       - `ajax`: `'sim'`
       - `ajaxOpcao`: `'publisher-pages-fetch'`
       - `params`: `{ publisher_id: publisherId, ids: schema.selected_items }`
     - Ao retornar `{ status: 'Ok', results: [...] }`, renderizar as tags dentro de `#selected-labels-container` respeitando estritamente a ordem contida em `schema.selected_items`.
  3. **Interface dos Labels (Tags)**:
     - Cada label inserido no contêiner deve possuir o formato de label do Fomantic UI com ícone de mover e botão de remover:
       ```html
       <div class="ui label teal drag-label" data-id="SLUG_DA_PAGINA" style="cursor: grab; display: inline-flex; align-items: center; user-select: none;">
           <i class="grip vertical icon" style="margin-right: 6px; opacity: 0.6;"></i>
           NOME_DA_PAGINA
           <i class="delete icon remove-tag-btn" style="margin-left: 8px; cursor: pointer;"></i>
       </div>
       ```
  4. **Comportamento do Campo de Busca (Autocomplete)**:
     - Escutar evento de `input` no campo `#manual_search_input`.
     - Aplicar um debounce de `300ms` (para evitar requisições AJAX excessivas a cada caractere).
     - Se o campo possuir texto, realizar requisição AJAX `publisher-pages-search` enviando:
       - `opcao`: `gestor.moduloOpcao`
       - `ajax`: `'sim'`
       - `ajaxOpcao`: `'publisher-pages-search'`
       - `publisher_id`: `publisherId`
       - `q`: texto digitado
     - Ao retornar `{ status: 'Ok', results: [...] }`, exibir a lista `#search-suggestions-dropdown`.
     - Cada linha retornada deve ser mapeada em uma div clicável (`.item`). Itens que já estão selecionados em `schema.selected_items` devem ser exibidos visualmente desabilitados (ou ignorados) e não permitir nova adição.
     - **Seleção de Item**: Ao clicar em uma sugestão, adicionar o slug ao final do array `schema.selected_items`, renderizar a nova tag correspondente no contêiner, limpar o campo de busca, ocultar a lista de sugestões, e atualizar o preview ao vivo (`scheduleWidgetPreview(false)`).
     - **Remoção de Item**: Ao clicar no ícone `.remove-tag-btn` de uma tag, removê-la do DOM, removê-la do array `schema.selected_items` e disparar `scheduleWidgetPreview(false)`.
     - **Fechamento do Dropdown**: Ocultar a lista flutuante se o usuário clicar fora do input/dropdown ou pressionar a tecla `Esc`.
  5. **Integração com Drag-and-Drop (Sortable.js)**:
     - Inicializar o `Sortable` no contêiner `#selected-labels-container` se a biblioteca global `Sortable` estiver disponível:
       ```javascript
       if (typeof Sortable !== 'undefined') {
           new Sortable(document.getElementById('selected-labels-container'), {
               animation: 150,
               handle: '.grip.vertical.icon',
               onEnd: function() {
                   // Relê a ordem física das tags no DOM e atualiza o schema
                   var newOrder = [];
                   $('#selected-labels-container .drag-label').each(function() {
                       var id = $(this).data('id');
                       if (id) newOrder.push(id);
                   });
                   schema.selected_items = newOrder;
                   scheduleWidgetPreview(false);
               }
           });
       }
       ```
  6. **Serialização e Envio do Formulário**:
     - No interceptador do submit do formulário (`$('.ui.form').on('submit', ...)`), certificar-se de serializar o `schema` atualizado como string JSON dentro do campo oculto `fields_schema`.
