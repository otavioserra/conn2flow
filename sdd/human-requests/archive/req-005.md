# Human Request: Renomeação Física de Diretórios e Arquivos de Templates (req-005)

Durante a integração do módulo `publisher-highlights` com o repositório principal, foi identificada uma divergência entre os identificadores (`id`) dos modelos no metadado `publisher-highlights.json` e os nomes de seus respectivos diretórios e arquivos físicos sob a pasta `resources/`.

Para que o motor de deploy e compilação do Conn2Flow funcione corretamente, os diretórios e arquivos físicos de templates devem possuir exatamente o mesmo nome do `id` declarado no JSON de metadados.

## Estratégia de Execução Sugerida

1.  **Script de Automatização (Opcional)**: O executor poderá criar um script rápido (ex: em Python) para ler os caminhos do `publisher-highlights.json`, localizar as pastas legadas e efetuar as renomeações e ajustes internos nos nomes dos arquivos de forma automática para evitar falha humana.
2.  **Limpeza de Checksums**: Limpar os hashes de checksum de todos os templates modificados no arquivo `publisher-highlights.json` (atribuindo string vazia `""` a `"html"`, `"css"` e `"combined"`).
3.  **Compilação de Recursos**: Rodar a tarefa de atualização do Core (`🗃️ Projects - Update => Core` ou o script php de compilação de recursos) para forçar o recálculo dos checksums corretos e o deploy no banco de dados e arquivos locais.
