# Human Request: Ajustes e Correções no Módulo de Menus (req-017)

Esta especificação define um lote corretivo (req-017) para resolver comportamentos anômalos no módulo `menus` (introduzido no req-016) e restaurar recursos de simulação/variáveis no Editor HTML.

---

## 1. Escopo das Modificações

### 1.1 Variáveis e Simulação no Editor HTML (menus.php & html-editor)
* **Contexto**: O módulo `publisher-highlights` possui uma aba/controle no editor HTML para alternar entre visualizar "Variáveis" e "Simular" a renderização com dados fictícios. Esta funcionalidade deve ser reintegrada para o módulo `menus`.
* **Ação**:
  - No backend (`menus.php`), configurar o editor HTML para o alvo `menus` para que exiba a aba de Simulação/Variáveis.
  - Garantir o carregamento das variáveis correspondentes aos menus (`[[item#label]]`, `[[item#url]]`, `[[item#css_classes]]`, `[[item#children]]`).

### 1.2 Componente de Simulação de Menus
* **Criação de Componentes**:
  - Criar o arquivo de componente de simulação para português: `gestor/resources/pt-br/components/html-editor-menus-simulation/html-editor-menus-simulation.html`
  - Criar o arquivo de componente de simulação para inglês: `gestor/resources/en/components/html-editor-menus-simulation/html-editor-menus-simulation.html`
* **Especificações**:
  - Estes componentes devem conter uma estrutura mockada de itens de menu simulados (incluindo sub-itens/filhos, links customizados, cabeçalhos, ações e separadores) para alimentar o preview simulado aleatório do editor HTML, assim como é feito para publicadores.

### 1.3 Correção do Alternador de Tipo de Item (menus.js)
* **Bug**: Ao selecionar outro tipo de item (ex: link-custom, cabecalho, link-action, separador) no dropdown de seleção do formulário de adição, os campos de entrada inferiores correspondentes (como inputs de Rótulo, URL, Classes CSS) não estão sendo exibidos, mantendo incorretamente exibida apenas a caixa de busca autocomplete ("Buscar Página").
* **Ação**:
  - Ajustar a lógica de escuta e alternância de classes/estilos em `menus.js` para ocultar a busca de páginas e exibir estritamente as caixas de texto corretas associadas ao tipo selecionado.

### 1.4 Melhorias no Placeholder do Drag-and-Drop (menus.js & CSS)
* **Placeholder visual (`menu-tree-row`)**:
  - A caixa tracejada azul (área de drop/soltura) que indica onde o item será encaixado durante o arraste bidimensional deve ser redimensionada para possuir a mesma altura do item de menu padrão (evitando que fique achatada verticalmente).
  - Adicionar uma indicação visual textual e simbólica dentro dessa área tracejada: exibir o texto `"Solte o item aqui"` (ou `"Drop item here"`) acompanhado de setas para a esquerda e para a direita (`←` / `→`), orientando o usuário de que ele pode recuar ou avançar o item para modular a hierarquia.

### 1.5 Correção de Foco/Hover nos Submenus (Preview / Templates)
* **Bug**: Nos templates que possuem submenus com abertura ao passar o mouse (hover), como no modelo `menus-dropdown`, quando o usuário move o ponteiro do mouse da opção principal em direção ao submenu aberto, a área suspensa desaparece.
* **Ação**:
  - Ajustar as propriedades de espaçamento/geometria CSS nos templates (evitando gaps vazios que quebram o estado de `:hover`) ou injetar rotinas em JavaScript na página de pré-visualização para reter o estado aberto enquanto o foco estiver navegando no elemento pai ou em seus descendentes.

---

## 2. Regras e Versionamento
* **ATUALIZAÇÃO DE RECURSOS**: Após realizar os ajustes, o script `atualizacao-dados-recursos.php` deve ser executado pelo executor.
* **CONTROLE DE VERSÃO**: Após a implementação deste lote de correções pelo executor, o operador realizará a homologação técnica.
