# req-055: Correção na Preservação do Estado de Disponibilidade do Menu no Carregamento do CRUD

Este lote de trabalho (BATCH-055) visa corrigir um bug de hidratação e restauração de estado no formulário de edição/clonagem do módulo `menus`. Atualmente, ao selecionar "Visibilidade Condicional", configurar as condições e salvar as alterações, o formulário recarrega a página exibindo incorretamente a opção de disponibilidade selecionada como "Visível a Todos" (reseta para o valor padrão).

---

## 1. Modificações na Interface Administrativa (JS de Menus)

### 1.1 Hidratação Correta do Campo "Disponibilidade do Menu" no Page Load
No arquivo `gestor/modulos/menus/menus.js`:
- Identificar o local onde o `schema` é carregado e os inputs do formulário são re-hidratados no carregamento inicial da página.
- Adicionar a inicialização para ler `schema.availability` do objeto JSON carregado do banco.
- Se `schema.availability` estiver presente (com o valor `todos` ou `condicional`):
  - Selecionar o valor correspondente no select de Disponibilidade do Menu (ex: `availability` ou o ID do select).
  - Atualizar o dropdown Fomantic-UI síncronamente (ex: `$('#availability').dropdown('set selected', schema.availability)`).
  - Acionar o fluxo correspondente para exibir/ocultar as seções corretas dentro de `#menu-availability` (carregar as abas de condições e a árvore ativa se for `condicional`, ou a árvore plana de menu se for `todos`).
- Garantir que o valor correto seja restaurado fielmente sem que o usuário precise alternar manualmente o select após reabrir a edição ou clonagem do menu.

---

## 2. Validação e Testes

1. **Testes Estáticos / Sintaxe**:
   - Rodar verificações estáticas de PHP e JS para validar a integridade dos arquivos:
     - `php -l gestor/modulos/menus/menus.php`
     - `node --check gestor/modulos/menus/menus.js`
2. **Validação Manual (Pendente com o operador)**:
   - Acessar a edição de um menu.
   - Trocar a disponibilidade para `Visibilidade Condicional`, cadastrar condições e clicar em "Salvar".
   - Confirmar se a página recarrega e exibe o select de disponibilidade com a opção `Visibilidade Condicional` selecionada e as abas das condições carregadas corretamente.
