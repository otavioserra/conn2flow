# Human Request: Sincronização de Projetos Específicos (Plano 1)

Este documento registra a demanda inicial para permitir que o fluxo de atualização (`🛠️ Manager - Update => All - Test Environment`) do core do Conn2Flow possa ser executado direcionado a projetos satélites/específicos cadastrados no `environment.json` (como o projeto `transformamp`).

## Requisitos principais descritos pelo Engenheiro Chefe:
1. **Identificador do Projeto**: Permitir passar o identificador do projeto (ex: `transformamp-local` ou `transformamp`) para ler suas credenciais e caminhos no arquivo `dev-environment/data/environment.json`.
2. **Sincronização de arquivos do Core**: Criar um mecanismo para transferir o que foi alterado no core (`conn2flow/gestor/`) para a pasta de desenvolvimento do projeto específico (ex: `/c/Users/otavi/OneDrive/Documentos/GIT/transformamp/gestor`).
3. **Atualização de Recursos e Banco de Dados**:
   - Rodar a compilação/atualização de recursos locais no escopo do projeto de destino.
   - Sincronizar o projeto com a área de testes no Docker.
   - Aplicar as atualizações de banco de dados locais apontando para o Docker path do projeto específico.
4. **Tarefas VS Code**: Agrupar tudo em tarefas automatizadas e fáceis de rodar via painel do VS Code.
