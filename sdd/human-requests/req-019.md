# Human Request: Novo Módulo de Galerias de Imagens (req-019)

Esta especificação define a criação do novo módulo de `galleries` (Galerias de Imagens), construído a partir de um clone estrutural do módulo `publisher-highlights`, adaptado para gerenciar e renderizar múltiplos arquivos de imagem selecionados do gerenciador de arquivos em lote.

---

## 1. Escopo das Modificações

### 1.1 Clonagem e Registro do Módulo `galleries`
* **Nomenclatura**: O identificador do módulo será `'galleries'` (Galerias de Imagens).
* **Clone de Destaques**: Clonar a estrutura do módulo `publisher-highlights` para criar o novo módulo.
* **Desacoplamento de Publisher**: Assim como foi feito no módulo de menus, a tabela e os controladores de `galleries` devem ser inteiramente livres de chaves ou regras relacionadas ao `publisher` (sem coluna `publisher_id`).
* **Banco de Dados**: Criar uma migração Phinx para a tabela `galleries` (id_galleries, name, fields_schema, html, css, status, language, versao, data_criacao, data_modificacao).
* **Registro de Módulos**: Registrar o novo módulo em `gestor/db/data/ModulosData.json` sob o grupo de administração gestor, com o ícone `"images"` e rótulos `"Galerias de Imagens"` / `"Image Galleries"`.

### 1.2 Curadoria e Seleção de Múltiplas Imagens (galleries.js)
* **Estrutura de Seleção**:
  - Em `galleries.js`, implementar a curadoria visual de imagens.
  - Um botão `"Selecionar Imagens do Servidor"` abrirá o modal de iframe padrão (`admin-arquivos`) usando a configuração do widget.
  - Adicionar um listener customizado para mensagens `message` (`postMessage`) vindas do iframe do gerenciador de arquivos.
  - Ao receber uma imagem, o script deve validar seu formato de imagem e adicioná-la à lista de itens selecionados da galeria.
  - **Diferencial de Seleção em Lote**: O script de recepção **não** deve fechar o modal automaticamente ao selecionar uma imagem. O modal deve permanecer aberto para permitir que o usuário clique em múltiplas imagens sequencialmente. O usuário fechará o modal de busca manualmente (clicando em "Fechar" ou fora do modal) quando tiver terminado.
* **Dados de Imagem na Lista**:
  - Cada imagem selecionada na lista deve exibir:
    - Um thumbnail visual da imagem (`imgSrc`).
    - O nome do arquivo original.
    - Um input de texto para que o usuário possa digitar uma **Legenda** personalizada para a imagem.
    - Um botão de remoção ("x") que retira a imagem do lote.
* **Reordenação Drag-and-Drop**:
  - Utilizar a biblioteca `Sortable.js` (incluída via CDN) para permitir a reordenação drag-and-drop das imagens na galeria.
* **Serialização do Schema**:
  - Salvar no input oculto `fields_schema` a lista final de imagens no formato:
    ```json
    {
      "template_id": "...",
      "selected_items": [
        {
          "id": "file_123456",
          "caminho": "caminho/para/imagem.jpg",
          "imgSrc": "url/para/imagem.jpg",
          "nome": "imagem.jpg",
          "legenda": "Minha legenda personalizada"
        }
      ]
    }
    ```

### 1.3 Backend: Popular e Integrar Modal de Imagem (galleries.php)
* **Integração do ImagePick**:
  - Integrar os dados de parametrização do widget de seleção de imagem em `galleries.php` nas funções de CRUD (`adicionar`, `editar` e `clonar`), permitindo que a janela modal do gerenciador de arquivos seja aberta.
  - Remover qualquer acoplamento de publishers ou curadorias automáticas (toda a alimentação de galerias é manual por imagem).

### 1.4 Renderização de Galerias no Widget (galleries.widget.php)
* **Lógica**:
  - Na função de renderização, decodificar `fields_schema` obtendo a lista de imagens de `selected_items`.
  - Processar os delimitadores `no-item` (se galeria estiver vazia) e `item`.
  - Para cada item de imagem, substituir as variáveis no delimitador `item`:
    - `[[item#img-src]]` -> URL completa da imagem para renderizar na tag `src`.
    - `[[item#caminho]]` -> Caminho relativo do arquivo.
    - `[[item#nome]]` -> Nome original do arquivo de imagem.
    - `[[item#legenda]]` -> Texto da legenda customizada.
* **Templates Padrão**:
  - Criar templates básicos de galeria de imagens na pasta `resources/{pt-br,en}/templates/` (ex: `galleries-grid`, `galleries-carousel`, `galleries-masonry`, `galleries-slider`) com blocos `item` e `no-item` para demonstrar o funcionamento em múltiplos formatos de exibição.

### 1.5 Simulação e Variáveis no Editor HTML
* **Lógica**:
  - Habilitar o painel de Simulação/Variáveis no editor HTML para o alvo `galleries`.
  - Criar os componentes de simulação mockados (`pt-br` e `en`) na pasta de recursos correspondente com uma lista fictícia de imagens contendo legendas, nomes e URLs do Picsum para alimentar a pré-visualização ao vivo.

## 2. Regras e Versionamento
* **RECURSOS**: Ao finalizar, executar a rotina de atualização de recursos (`atualizacao-dados-recursos.php`) para compilar e registrar os novos templates e o módulo de galerias.
* **RESTRIÇÃO**: Sem commits ou pushes automáticos no Git; as alterações permanecem locais no executor.
