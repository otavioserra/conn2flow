# Human Request: Correções no Menus e Lógica do Módulo de Galerias (req-019)

Esta especificação define o lote de trabalho `req-019` (BATCH-019) para o Engenheiro Executor. O foco é corrigir problemas de design e layout reportados no módulo de `menus`, implementar o suporte a `target` em links personalizados, divisor visual nos separadores, a correção de resolução de imagens no módulo `galleries`, controles de carrossel (setas, dot pagination, autoplay) e a lógica de script dinâmico dos widgets públicos (`menus.widget.js` e `galleries.widget.js`).

---

## 1. Módulo de Menus

### 1.1 Interface de Criação e Edição (HTML / JS / PHP)
* **Espaçamento**: Adicionar espaçamento de `margin-top: 1rem;` no contêiner do botão `#btn-add-item-wrapper` em todos os formulários (`menus-adicionar.html`, `menus-editar.html`, `menus-clonar.html` em `pt-br` e `en`) para evitar que fique colado na caixa de ordenação do publicador.
* **Alvo de Link Personalizado (`target`)**:
  - Adicionar o campo dropdown de seleção de target (`#custom-target`) contendo as opções `_self` (Mesma janela) e `_blank` (Nova aba).
  - Exibir o campo alvo (`#field-custom-target`) condicionalmente apenas quando o tipo selecionado for `link-custom`.
  - Sincronizar o estado de salvamento desse campo no schema `selected_items` para itens do tipo `link-custom`.
  - Suportar a edição inline deste campo na árvore visual (dropdown `edit-target`).
* **Rótulo no Separador**:
  - Exibir o campo de Rótulo do Menu (`#field-custom-label`) quando o tipo selecionado for `separador` para permitir títulos ou textos divisores opcionais.
  - Sincronizar o estado e permitir a edição inline deste rótulo na árvore visual.
* **Mapeamento de Variáveis**:
  - Adicionar a variável `target` à função `menus_variaveis_template()` em `menus.php`.

### 1.2 Widget Renderer e Templates (`menus.widget.php` & Templates)
* **Novo Delimitador `item-separator`**:
  - Em `menus.widget.php`, extrair o bloco `item_separator` a partir da tag `<!-- item-separator < --> ... <!-- item-separator > -->`.
  - Na recursão `menus_render_level()`, se o item for do tipo `separador` e o template definir `item_separator`, renderizar este bloco (passando variáveis como `[[item#label]]`). Caso não exista o delimitador, cair no fallback anterior.
* **Resolução de Link Target**:
  - Repassar a variável `[[item#target]]` ao renderizar os links de `item` e `item-parent` em `menus.widget.php` (padrão `_self` para páginas e links, vazio para separadores).
  - Atualizar todos os 12 templates para incluir `target="@[[item#target]]@"` nas tags de link `<a>`.
* **Bloco Separador nos Templates**:
  - Incluir a marcação do divisor físico `<!-- item-separator < --> ... <!-- item-separator > -->` em todos os 12 templates, formatados conforme cada layout (ex: `<hr class="border-t border-gray-200 my-1">` em dropdown/sidebar ou uma barra/espaçador em menus horizontais).
* **Ajustes Visuais de Layout**:
  - `menus-horizontal-navbar`: Aumentar o espaçamento horizontal no `ul` de `gap-6` para `gap-8`.
  - `menus-mobile-hamburguer`: Trocar a lógica do checkbox toggle peer por um botão interativo `.menu-mobile-btn` e adicionar a classe `.menu-mobile-list` na lista `ul` de itens para controle no JavaScript do widget.
  - `menus-footer-colunas`: Substituir a tag `<span>` do título do bloco `item-parent` por uma tag `<a>` com `href="@[[item#url]]@"` e `target="@[[item#target]]@"`, permitindo cliques no elemento pai que possui URL.
  - `menus-dropdown`: Adicionar a classe `.group-hover-sub-trigger` no `li` do `item-parent` e `.group-hover-sub-menu` no `ul` do submenu para fallback de exibição manual.

### 1.3 JavaScript do Widget (`menus.widget.js`)
* **Hamburger Menu**: Adicionar listener de clique em `.menu-mobile-btn` que alterna a exibição (classes `hidden` / `flex`) da lista `.menu-mobile-list` correspondente.
* **Dropdown Hover Fallback**: Implementar detecção de mouseenter/mouseleave em `.group-hover-sub-trigger` que alterna a exibição (classes `hidden` / `block`) do submenu `.group-hover-sub-menu` adjacente, prevenindo falhas caso named groups do Tailwind CSS não sejam suportados no ambiente de destino.

## 2. Módulo de Galerias de Imagens

### 2.1 Interface Administrativa (HTML / JS)
* **Parâmetros dos Controles**:
  - Nos formulários (`galleries-adicionar.html`, `galleries-editar.html`, `galleries-clonar.html` em pt-br e en), adicionar os campos de configurações de controle:
    - Exibir Setas (`show_arrows` - checkbox, padrão ativo).
    - Exibir Pontinhos (`show_dots` - checkbox, padrão ativo).
    - Slide Automático (`autoplay` - checkbox, padrão inativo).
    - Tempo de Transição (`autoplay_speed` - input numérico, padrão 3000ms, mínimo 500ms).
    - Loop Infinito (`loop` - checkbox, padrão ativo).
  - Hidratar esses campos a partir do schema e incluí-los na serialização e preview do snapshot em `galleries.js`.

### 2.2 Resolução e Renderização Pública (`galleries.widget.php`)
* **Resolução da Imagem Original**:
  - Em `galleries_widget_resolver_item_vars()`, priorizar o campo `caminho` (original) sobre `imgSrc` (thumbnail) para montar o endereço da imagem pública `img-src`, prefixando com a url-raiz caso seja relativo. O `imgSrc` continua sendo usado no painel de controle (thumbnails leves).
* **Parâmetros de Controle no DOM**:
  - Injetar no elemento de contêiner principal da galeria atributos de dados contendo os valores parametrizados (ex: `data-autoplay="true" data-speed="3000" data-loop="true"`).
* **Controles Dinâmicos no Widget**:
  - Em `galleries.widget.php`, processar as regiões de controles dinâmicos:
    - `<!-- controls-arrows < --> ... <!-- controls-arrows > -->`: Renderizado apenas se `show_arrows` for verdadeiro.
    - `<!-- controls-dots < --> ... <!-- controls-dots > -->`: Renderizado apenas se `show_dots` for verdadeiro.
    - `<!-- dot-item < --> ... <!-- dot-item > -->`: Repetido para cada imagem no lote de curadoria, injetando `[[dot#index]]` (0, 1, 2, ...).

### 2.3 Templates de Galerias (`galleries-carousel` & `galleries-slider`)
* **Modificações de Estrutura**:
  - Adicionar os delimitadores condicionais de setas e dots nos templates deslizantes.
  - Adicionar classes identificadoras para a interação com o script JS (ex: `.conn2flow-gallery`, `.gallery-slides-wrapper`, `.gallery-slide`, `.gallery-prev`, `.gallery-next`, `.gallery-dots`, `.gallery-dot`).

### 2.4 JavaScript do Widget (`galleries.widget.js`)
* **Comportamento Interativo**:
  - Para cada `.conn2flow-gallery` com múltiplos itens, inicializar a lógica do carrossel/slider:
    - Ler as propriedades `data-autoplay`, `data-speed`, `data-loop` no contêiner.
    - Implementar a navegação suave alterando o `scrollLeft` do `.gallery-slides-wrapper` via animação jQuery `.animate({ scrollLeft: ... })`.
    - Tratar cliques nas setas `.gallery-next` e `.gallery-prev` (fazendo wrap/loop se configurado).
    - Tratar cliques nos dots `.gallery-dot`, sincronizando o slide e a classe ativa do pontinho.
    - Implementar e gerenciar o timer de auto-play (`setInterval`), limpando e reiniciando a contagem sempre que houver intervenção manual do usuário.
