# req-072: Remoção do Campo e Metadado "hosting_plan" no Módulo de Planos de Assinatura (lumix)

## Descrição do Problema / Contexto
O campo `hosting_plan` (Plano de Hospedagem) foi criado no banco de dados e na interface administrativa de planos de assinatura (`subscriptions-plans`). No entanto, ele não possui nenhuma utilidade operacional ou lógica ativa no sistema (trata-se de um campo sem uso no backend e no frontend do checkout público).

Para evitar complexidade desnecessária e manter a base de código limpa, queremos remover por completo a coluna `hosting_plan` do banco de dados, da interface do CRUD de Planos e da hidratação de configurações no módulo `subscriptions` do projeto **`lumix`**.

---

## Mudanças Necessárias (Repositório `lumix`)

### 1. Banco de Dados e Migração (`lumix`)
- Criar uma nova migração Phinx no diretório `gestor/db/migrations/` (exemplo: `20260710130000_remove_hosting_plan_from_subscriptions_plans.php`) para remover com segurança a coluna `hosting_plan` da tabela `subscriptions_plans`.
- A migração deve possuir um guard `hasColumn` e remover a coluna no método `up()` e, opcionalmente, recriá-la no método `down()`.

---

### 2. Interface Administrativa de Planos (`subscriptions-plans` em `lumix`)

#### 2.1. Remoção no Controller (`gestor/modulos/subscriptions-plans/subscriptions-plans.php`)
- Remover toda menção à chave/coluna `'hosting_plan'`:
  - Na lista de atribuição de parâmetros do request à variável `$campos` (função de inserção/atualização).
  - No array de campos permitidos do banco (`$camposBanco`).
  - No laço `foreach` que valida/processa os campos textuais enviados.
  - Na montagem dos placeholders do template (`#hosting_plan#`).

#### 2.2. Remoção nos Templates HTML de CRUD
- Nos 4 templates de página em `gestor/modulos/subscriptions-plans/resources/`:
  - `pt-br/pages/subscriptions-plans-adicionar/subscriptions-plans-adicionar.html`
  - `pt-br/pages/subscriptions-plans-editar/subscriptions-plans-editar.html`
  - `en/pages/subscriptions-plans-adicionar/subscriptions-plans-adicionar.html`
  - `en/pages/subscriptions-plans-editar/subscriptions-plans-editar.html`
- Remover o campo `<input type="text" name="hosting_plan">` e todo o container `<div class="field">...</div>` ou estrutura HTML correspondente que o envolve.

---

### 3. Hidratação de Configurações (`subscriptions` em `lumix`)

#### 3.1. Remoção no Hooks (`gestor/modulos/subscriptions/subscriptions.hooks.php`)
- Na função `subscriptions_hidratar_config_bd(&$config)`:
  - Remover `'hosting_plan'` do array de colunas consultadas na tabela `subscriptions_plans` via `banco_select`.
  - Remover o bloco condicional de atribuição à variável de configuração:
    ```php

    ```
