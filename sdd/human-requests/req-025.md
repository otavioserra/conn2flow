# Human Request: Autocomplete de Páginas em Galerias, Ajuste do Menu Horizontal e Preparação Final de Release (req-025)

Esta especificação define o lote de trabalho `req-025` (BATCH-025) para o Engenheiro Executor. O escopo abrange a incorporação do buscador AJAX com autocomplete de páginas na curadoria individual de imagens de galerias, correção de links inativos no widget de galeria, alinhamento da seta indicadora de submenu no navbar, ampliação das miniaturas de imagem curadas, e a atualização final dos arquivos de documentação e pipeline de CI/CD para o lançamento da versão 2.8.0.

---

## 1. Contexto e Requisitos

### 1.1 Autocomplete de Páginas em Galerias
*   Na curadoria de imagens, ao configurar um link do tipo **Página**, o painel atualmente exibe um dropdown `<select>` simples com todas as páginas do site. Para sites com grande volume de páginas, isso é inviável.
*   Deve-se replicar o comportamento do módulo **Menus**: ao selecionar o tipo de link **Página**, exibir dinamicamente o filtro de tipo de página (Página, Sistema, Ambos) e um input de busca que faz autocomplete AJAX a partir do servidor.
*   Como cada item da lista curada é dinâmico e existem múltiplas linhas abertas na curadoria, os elementos (inputs, checkboxes, listas de sugestão) devem ser diferenciados via classes ou IDs contendo o `id` único do item para evitar conflitos de seleção jQuery.
*   Vincular os endpoints AJAX de busca (`pages-search`) e hidratação (`pages-fetch`) no controlador PHP do módulo de galerias.

### 1.2 Links Inativos nas Galerias
*   No widget público e no previewer, as imagens da galeria estão sendo envolvidas por tags `<a>`. Se o usuário não configurou nenhum link (tipo `'nenhum'`), o cursor do mouse ainda muda para ponteiro ("dedinho") e as imagens parecem clicáveis.
*   Quando o link não estiver configurado (`link_type === 'nenhum'`), a imagem não deve disparar eventos de clique nem alterar o cursor visual. A classe utilitária do Tailwind `pointer-events-none cursor-default` deve ser adicionada dinamicamente à tag `<a>` nestes casos.

### 1.3 Alinhamento de Seta de Submenu no Navbar Horizontal
*   No template `menus-horizontal-navbar`, o espaçamento e padding dos submenus recursivos (níveis 2+) foram corrigidos, mas os itens pai desses níveis exibem o caractere/SVG da setinha quebrado na linha abaixo do rótulo, em vez de na mesma linha à direita.
*   Isso ocorre porque o seletor `[data-title="menu-horizontal-navbar"] ul.absolute a` força `display: block !important`.
*   O seletor deve ser ajustado para usar `display: flex !important; align-items: center; justify-content: space-between;` (ou similar) garantindo o alinhamento horizontal em uma única linha.

### 1.4 Aumento do Tamanho do Thumb da Imagem
*   Na lista curada de imagens do painel administrativo de galerias, a classe `.gallery-item-thumb` possui dimensões estáticas de `64px` por `48px`.
*   Para melhorar a identificação das fotos curadas, a dimensão deve ser ampliada para `200px` de largura por `140px` de altura.

### 1.5 Atualização de READMEs, CHANGELOGs e Workflow de Release
*   Atualizar a data de lançamento da versão `2.8.0` para a data de hoje nos arquivos `CHANGELOG.md` e `CHANGELOG-PT-BR.md`.
*   Incluir as melhorias de otimização de CSS redundante (BATCH-023), links individuais em galerias, exibição dinâmica e correções de layout (BATCH-024 e BATCH-025) nos changelogs.
*   Atualizar os arquivos `README.md` e `README-PT-BR.md` com a documentação do novo módulo de Menus, Galerias e compilação CLI do Tailwind CSS v4.

Como o array global `galleries_pages` já está carregado em memória, fazer uma varredura local para encontrar o nome e preencher o input de busca. Caso não encontre, fazer um fetch via AJAX `pages-fetch`.
