# Human Request: Tipo de Item Publicador no Módulo de Menus (req-018)

Esta especificação define a inclusão do novo tipo de item de menu "Publicador" (`publicador`) no módulo `menus`. Esse item funciona como um nó gerador que carrega dinamicamente páginas do publicador selecionado do banco de dados no momento de renderizar o menu.

---

## 1. Escopo das Modificações

### 1.1 Interface de Entrada no Formulário (Páginas HTML × pt-br/en)
* **Novo Tipo no Dropdown**:
  - Adicionar a opção "Publicador" (ou "Publisher") no dropdown `#menu_item_type`.
* **Novos Campos Condicionais**:
  - Exibir os campos abaixo apenas quando o tipo "Publicador" for selecionado:
    - **Seletor de Publicador**: Dropdown populado dinamicamente com `#publisher_id_options#` (carrega os registros da tabela `publisher`).
    - **Limite**: Input numérico para definir a quantidade máxima de itens filhos (`count`, padrão `5`, mínimo `1`).
    - **Ordenação**: Dropdown com opções para definir a ordem das páginas do publicador:
      - `date_desc`: Por data (mais recentes primeiro)
      - `date_asc`: Por data (mais antigas primeiro)
      - `title_asc`: Alfabética (A-Z)
      - `title_desc`: Alfabética (Z-A)

### 1.2 Comportamento da Árvore de Menus (menus.js)
* **Estrutura no Schema**:
  - Salvar no schema os dados do item tipo `publicador`: `publisher_id`, `count` e `order_by` (além de `id`, `type`, `label` e `css_classes`).
* **Visualização da Árvore**:
  - Renderizar na árvore visual a barra do item indicando o tipo (ex: `Publicador: Notícias (limite: 5)`).
  - Impedir que o usuário arraste ou insira manualmente sub-itens sob um nó do tipo `publicador` (ou avisar que eles serão mesclados/ignorados), uma vez que os seus filhos reais são dinâmicos.
  - No painel inline de edições do item da árvore, exibir as opções corretas para alterar o Rótulo, o Publicador selecionado, o Limite (`count`), a Ordenação (`order_by`) e as Classes CSS.

### 1.3 Backend: Popular Publicadores Registrados (menus.php)
* **Lógica**:
  - Implementar a função `menus_publisher_options($selected_id)` para buscar os registros da tabela `publisher` (filtrando por status ativo e linguagem corrente) e substituir `#publisher_id_options#` no HTML.
  - Invocar esta função nos métodos de CRUD (`menus_adicionar`, `menus_editar` e `menus_clonar`).

### 1.4 Backend: Resolução de Filhos em Runtime (menus.widget.php)
* **Lógica**:
  - Implementar em `menus.widget.php` a função `menus_widget_buscar_paginas_publisher($publisher_id, $count, $order_by)` para buscar as publicações ativas da tabela `paginas` para o idioma corrente filtradas pelo `publisher_id`.
  - Na função de renderização inline, inspecionar a árvore e, antes de renderizar recursivamente, preencher a chave `children` dos nós de tipo `publicador` com as páginas resultantes da busca acima (como nós dinâmicos temporários do tipo `pagina`).

### 1.5 Simulação no Editor HTML (html-editor-interface.js & simulation htmls)
* **Lógica**:
  - Adaptar a simulação recursiva de visualização de menus: ao encontrar um nó do tipo `publicador`, injetar dinamicamente `$count` sub-itens simulados de tipo `pagina` na árvore em memória para que o editor exiba o preview com filhos fictícios da forma correta.
  - Atualizar os arquivos de componente de simulação de menus (`pt-br` e `en`) para incluir um exemplo demonstrativo de item `publicador` na árvore mockada.

### 1.6 Correções Adicionais no Módulo de Menus (menus.js & templates)
* **Ajuste no Template ID**:
  - Quando a opção de modelo visual `template_id` for alterada, garantir que o conteúdo HTML/CSS do CodeMirror do Editor HTML seja atualizado corretamente, mesmo se a aba do editor estiver focada no momento da mudança.
* **Ajuste no Alternador de Tipo de Item (`item_type`)**:
  - Corrigir a lógica de alternar campos de entrada do formulário. Quando trocar para link personalizado, cabeçalho, separador, etc., exibir apenas as caixas correspondentes (ex: `add-field-label`, `add-field-url`) e ocultar as opções de busca de páginas.
* **Inclusão de Variáveis nos Templates**:
  - Integrar os placeholders de variáveis `[[item#slug]]` e `[[item#css_classes]]` (escritos como `@[[item#slug]]@` e `@[[item#css_classes]]@` no banco/templates) em todos os 12 templates do módulo `menus` (em `pt-br` e `en`) para que as classes e slugs sejam renderizados no HTML final.
