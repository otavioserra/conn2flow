# Human Request: Correção de Sincronização e Race Conditions no Dropdown de Curadoria (req-013)

Esta especificação detalha os ajustes necessários no dropdown customizado de curadoria manual (`jquery-custom-dropdown.js`) e no módulo de destaques (`publisher-highlights.js`) para corrigir o bug de pré-visualização que deixa de atualizar ou exibe apenas o primeiro item após a adição de múltiplos cards ou ao remover a seleção.

---

## Detalhes do Bug e da Causa Raiz

### O Problema
Ao interagir com o dropdown de itens selecionados no modo de curadoria manual:
1. Ao adicionar o segundo item, a pré-visualização não é atualizada e continua exibindo apenas o primeiro card.
2. Ao remover itens (clicando no ícone "x" da tag), a pré-visualização para de funcionar e não limpa/atualiza o estado.
3. Ao alternar o template visual com itens selecionados, a renderização exibe apenas o primeiro item.

### A Causa Raiz
No lote anterior, implementamos callbacks `onAdd` e `onRemove` no Fomantic UI para rastrear a ordem cronológica de cliques no array `this.selectedIds`. No entanto:
1. Métodos como `$dd.dropdown('clear', true)` e `$dd.dropdown('set exactly', ...)` chamados programaticamente durante buscas (autocompletar) ou carregamento (hidratação) em `setValues` também disparam os callbacks de evento `onRemove` e `onAdd` do Fomantic UI.
2. Isso causa um loop de atualizações concorrentes e assíncronas que corrompem o array `this.selectedIds` e chamam `setSelectedIds` repetidas vezes com dados parciais ou desatualizados.
3. O método `syncSelection()` é disparado por eventos de clique/mudança nativos e tenta ler a seleção crua do DOM (`readSelection()`) quando o array interno é limpo (tamanho 0). Devido a atrasos de atualização do DOM do select, ele lê e restaura o item deletado, dessincronizando a UI e impedindo o disparo de novos previews (porque o snapshot do preview identifica que o schema "não mudou").

---

## Correções Solicitadas

### 1. Prevenção de Concorrência de Callbacks em Atualizações Programáticas
* **Local**: `gestor/assets/interface/jquery-custom-dropdown.js`
* **Ajuste**:
  - Adicionar a propriedade/flag `this.ignoreCallbacks = false` no construtor da classe `PublisherHighlightsCustomDropdown`.
  - No método `setValues`, envelopar todas as chamadas de alteração do dropdown Fomantic UI (`change values`, `clear`, `set exactly`) definindo a flag `this.ignoreCallbacks = true` no início e `this.ignoreCallbacks = false` no final:
    ```javascript
    this.ignoreCallbacks = true;
    // ... operações no dropdown ...
    this.ignoreCallbacks = false;
    ```
  - Nos callbacks `onAdd` e `onRemove` definidos em `settings()`, verificar a flag no início de cada execução e retornar imediatamente caso seja verdadeira, impedindo qualquer mutação ou propagação de preview para chamadas de API:
    ```javascript
    onAdd: function (addedValue) {
        if (self.ignoreCallbacks) return;
        // ... lógica existente ...
    },
    onRemove: function (removedValue) {
        if (self.ignoreCallbacks) return;
        // ... lógica existente ...
    }
    ```

---

### 2. Proteção e Simplificação de `syncSelection`
* **Local**: `gestor/assets/interface/jquery-custom-dropdown.js`
* **Ajuste**:
  - No método `syncSelection()`, retornar imediatamente se `this.ignoreCallbacks` for verdadeiro.
  - Como o array interno `this.selectedIds` é inicializado no construtor e hidratado logo no carregamento em `setValues`, simplificar a lógica de `syncSelection` para sempre confiar e propagar o array `this.selectedIds` fatiado, eliminando completamente a leitura e reidratação a partir de `readSelection()` quando o array fica vazio. O leitor cru do DOM só deve ser utilizado na inicialização se o estado do array não existir:
    ```javascript
    PublisherHighlightsCustomDropdown.prototype.syncSelection = function () {
        if (this.ignoreCallbacks) return;
        var selectedIds = (this.selectedIds || []).slice();
        this.options.setSelectedIds(selectedIds);
    };
    ```

---

### 3. Validação do Acoplamento de Preview
* **Local**: `gestor/modulos/publisher-highlights/publisher-highlights.js`
* **Ajuste**:
  - Validar que o callback `setSelectedIds` repassa corretamente o array ordenado de IDs para `schema.selected_items` e que `scheduleWidgetPreview(false)` é invocado com sucesso, forçando o processamento dinâmico sem restrições.
