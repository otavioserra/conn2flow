# Human Request: Ajuste na Tarefa Composta de Sincronização (req-002)

Ao executar a nova tarefa de sincronização de projetos, identificou-se que as modificações de recursos (layouts, páginas, componentes) feitas no core (`conn2flow/gestor/`) não são compiladas antes do core ser sincronizado. Isso faz com que arquivos JSON de recursos legados/desatualizados no core sejam copiados para a pasta de testes do Docker do projeto.

## Ajustes Solicitados:
1. **Nova Task no VS Code**: Criar a tarefa `"🗃️ Projects - Resources Core -> ID"` que executa a atualização de recursos locais no core (`atualizacao-dados-recursos.php`).
2. **Atualização da Task Composta**: Atualizar a tarefa `"🗃️ Projects - Update => All - Test Environment"` (renomeando-a para usar o prefixo `🗃️` em vez de `🛠️`) para incluir esta nova etapa de compilação de recursos do core como o primeiro passo do fluxo:
   - `"🗃️ Projects - Resources Core -> ID"`
   - `"🗃️ Projects - Sync Core -> ID"`
   - `"🗃️ Projects - Synchronize => Resources -> ID"`
   - `"🗃️ Projects - Synchronize => Files -> ID"`
   - `"🗃️ Projects - Synchronize => Database -> ID"`
