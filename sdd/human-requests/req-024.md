# Human Request: Links Dinâmicos em Galerias, Controles de Exibição e Correções de Layout (req-024)

Esta especificação define o lote de trabalho `req-024` (BATCH-026) para o Engenheiro Executor. O escopo abrange a implementação de links configuráveis nas imagens de galerias, controles dinâmicos de altura e margem nas galerias, e correções de layout e espaçamento nos módulos de Destaques, Menus e Galerias.

---

## 1. Contexto e Requisitos

### 1.1 Links Personalizados por Imagem em Galerias
No módulo de Galerias (`galleries`), cada imagem curada na lista exibe apenas o thumbnail, nome do arquivo e campo de legenda. É necessário tornar as imagens clicáveis individualmente na galeria renderizada.
* Cada item da curadoria de imagens passará a aceitar a configuração opcional de um link, com os seguintes tipos de destino:
  1. **Página:** Seleção de uma página interna do site (gera o link canônico correspondente).
  2. **Link Customizado:** Inserção manual de URL e seleção do alvo (`_self` ou `_blank`).
  3. **Link com Classe CSS:** Inserção manual de URL e classe CSS customizada.
  4. **Última Publicação (Publicador):** Seleção de um Publicador de Destaques ativo; no carregamento do site, o link resolverá dinamicamente para a URL da publicação mais recente desse publicador (com ordenação selecionável).
* Na interface do painel, cada linha de imagem na lista terá um painel retrátil "Configurar Link" contendo os dropdowns e inputs necessários, os quais aparecerão de acordo com o tipo de link selecionado.
* Os templates de galeria serão atualizados para dar suporte às novas variáveis: `[[item#link-url]]`, `[[item#link-target]]` e `[[item#link-css-classes]]`.

### 1.2 Controles de Exibição de Galerias (Altura e Margem Lateral)
Para melhorar a adequação visual das galerias nas páginas:
* Inclusão de dois novos campos numéricos na aba "Controles de Exibição" no painel de Galerias:
  1. **Altura do Container:** Altura vertical em pixels. Padrão: `300`.
  2. **Margem Lateral:** Distância em pixels nas laterais esquerda e direita da galeria. Padrão: `0` (encostado).
* Os templates visuais de galerias (carrossel, slider, grid) devem ser atualizados para aplicar dinamicamente esses valores no estilo dos elementos (imagens e seções raiz), utilizando as variáveis globais de controle `[[height]]` e `[[margin_lateral]]`.

### 1.3 Correção de Mapeamento no Destaques (`linked-fields-list`)
No mapeamento de variáveis do módulo de Destaques (`publisher-highlights`), as tags de variáveis vinculadas no container `#linked-fields-list` estão coladas horizontalmente. É necessário espaçá-las adicionando margens horizontais.

### 1.4 Correção do Menu Horizontal (`menus-horizontal-navbar`)
No template `menus-horizontal-navbar` (tanto em pt-br quanto en), os submenus (nível 2 e subsequentes) estão sem recuo horizontal (padding) interno, fazendo com que o texto dos links encoste na borda esquerda da caixa flutuante. É necessário aplicar o margeamento/padding adequado via CSS nos links internos das camadas inferiores do dropdown.

### 1.5 Correção do Layout de Alvenaria (`galleries-masonry`)
No template `galleries-masonry` (tanto em pt-br quanto en), a legenda das imagens está muito próxima das bordas esquerda e inferior. É necessário aumentar esse espaçamento aplicando margeamento ou padding.
