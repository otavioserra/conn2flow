## Descrição do Problema / Contexto
Este lote consolidado de modificações agrupa três grandes metas de evolução do ecossistema `conn2flow` relacionadas com a navegação administrativa, agendamento de publicações e edição em tempo de execução:

1. **Acessar Site do Layout Administrativo**: Adicionar um botão no cabeçalho do layout administrativo para facilitar a navegação rápida do administrador de volta para a raiz pública do site.
2. **Dashboard Site Toolbar (Iframe)**: Uma barra superior flutuante de ~30px (estilo WordPress admin bar) injetada em páginas públicas para usuários logados com permissão de edição. Ela abrigará botões para:
   * Edição visual/live da página HTML atual.
   * Atalho inteligente de edição avançada no painel administrativo (redirecionando para `publisher-pages` ou `admin-paginas` dependendo do tipo da página).
3. **Agendamento e Datas Customizadas**: Permitir nos formulários do `admin-paginas` e `publisher-pages` o agendamento de visibilidade (início e fim da publicação) e a alteração retroativa das datas de criação (`data_criacao`) e modificação (`data_modificacao`).
4. **Atalhos de Gestão de Widgets no Editor HTML**: Permitir no editor visual de HTML (tanto no novo via toolbar quanto no clássico dos módulos) que o usuário acesse diretamente a página administrativa de edição do widget selecionado via botão específico para isso (ex: ir para o módulo `menus` ao selecionar um widget do tipo menu).

### Meta 4: Atalho de Edição Avançada Inteligente
* **Especificação**:
  * Adicionar o botão **"Editar Página Avançado"** na Toolbar.
  * Lógica de Redirecionamento:
    * Se a página ativa do site foi gerenciada/gerada pelo módulo do publicador de páginas (verificando no banco de dados na tabela `paginas` se possui metadados ou flag de `publisher_id`), o botão deve abrir a página de edição do módulo `publisher-pages`.
    * Caso contrário (página comum do sistema), abrir o módulo `admin-paginas` diretamente no registro em questão.
    * Use o campo `id` e `publisher_id` para acessar a página no módulo correto, formando a URL de edição como:
      * `admin-paginas/editar/?id={id}` ou
      * `/publisher-pages/editar/?id={publisher_id}`.
  * O link usará `target="_parent"` para abrir no navegador principal.
