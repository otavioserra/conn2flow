# req-051: Escape HTML Global no Editor, Divisão de Opções e Tipos Radio/Checkbox no Módulo Forms

Este lote de trabalho (BATCH-051) define a migração do escape de HTML para comportamento padrão no `html-editor` (quando `html` e `html_extra_head` são informados), implementa a divisão de `value` e `label` para os campos do tipo select no renderizador público, e adiciona suporte completo a campos do tipo `radio` e `checkbox` no módulo Forms (com exibição condicional do campo Opções no CRUD, placeholder instrutivo e suporte nos templates).

## 2. Divisão de Opções e Suporte a Radio/Checkbox (Forms)

### 2.1 Modificar `gestor/modulos/forms/forms.widget.php`
- Atualizar a função `forms_widget_options_html($field)` para:
  - Recuperar a chave de idioma ou tipo de campo `$field['type']`.
  - No loop `foreach ($options as $option)` do bloco `else` (quando processa opções declaradas via string de linha), validar a presença dos separadores `:` ou `|` para separar a chave (`value`) da descrição (`label`).
  - Se o tipo de campo for `radio` ou `checkbox`, renderizar elementos HTML do tipo `<input>` dentro de um wrapper `<label>` em vez de tags `<option>`.
- **Implementação recomendada**:
```php
function forms_widget_options_html($field) {
	$options = $field['options'] ?? [];
	$type = $field['type'] ?? 'select';
	$name = $field['name'] ?? '';
	$required = forms_widget_bool($field['required'] ?? false) ? 'required' : '';

	if (is_string($options)) {
		$options = array_filter(array_map('trim', preg_split('/\r\n|\r|\n|,/', $options)));
	}
	if (!is_array($options)) return '';

	$html = '';
	foreach ($options as $option) {
		if (is_array($option)) {
			$value = $option['value'] ?? ($option['label'] ?? '');
			$label = $option['label'] ?? $value;
		} else {
			if (strpos($option, '|') !== false) {
				$parts = explode('|', $option, 2);
				$value = trim($parts[0]);
				$label = trim($parts[1]);
			} else if (strpos($option, ':') !== false) {
				$parts = explode(':', $option, 2);
				$value = trim($parts[0]);
				$label = trim($parts[1]);
			} else {
				$value = $option;
				$label = $option;
			}
		}

		if ($type === 'radio' || $type === 'checkbox') {
			$input_name = htmlspecialchars((string)$name, ENT_QUOTES, 'UTF-8') . ($type === 'checkbox' ? '[]' : '');
			$html .= '<label class="inline-flex items-center mr-4 mb-2 cursor-pointer">'
				. '<input type="'.$type.'" name="'.$input_name.'" value="'.htmlspecialchars((string)$value, ENT_QUOTES, 'UTF-8').'" class="rounded border-gray-300 text-indigo-600 focus:ring-indigo-500" '.$required.'>'
				. '<span class="ml-2 text-sm text-gray-700">'.htmlspecialchars((string)$label, ENT_QUOTES, 'UTF-8').'</span>'
				. '</label>';
		} else {
			$html .= '<option value="'.htmlspecialchars((string)$value, ENT_QUOTES, 'UTF-8').'">'.htmlspecialchars((string)$label, ENT_QUOTES, 'UTF-8').'</option>';
		}
	}
	return $html;
}
```

- Na função `forms_widget_render_field()`, atualizar as condicionais de bloco para suportar `type-radio` e `type-checkbox`, e isolar a condicional de `type-input`:
```php
	$html = forms_widget_block($html, 'type-select', $type === 'select');
	$html = forms_widget_block($html, 'type-textarea', $type === 'textarea');
	$html = forms_widget_block($html, 'type-radio', $type === 'radio');
	$html = forms_widget_block($html, 'type-checkbox', $type === 'checkbox');
	$html = forms_widget_block($html, 'type-input', $type !== 'select' && $type !== 'textarea' && $type !== 'radio' && $type !== 'checkbox');
```
