## What are Module Groups?

**Module Groups** organize modules into logical categories on the Dashboard. Groups help users find related modules quickly and keep the interface organized.

---

## üéØ Getting Started

### Accessing Module Groups
1. On the Dashboard, find the **Module Groups** card
2. Click to open the module
3. You'll see all defined groups

> üîí This is an administrator area. You need admin permissions.

---

## üìã Group List

### What You'll See
For each group:
- **Name** - Group display name
- **ID** - Unique identifier
- **Order** - Display position
- **Modules** - Count of modules in group
- **Actions** - Edit, reorder, delete

### Default Groups
| Group | Contains |
|-------|----------|
| **Administration** | System settings, updates |
| **Publisher** | Content management |
| **Users** | User management |
| **Modules** | Module configuration |

## ‚ûï Creating Groups

### How to Create
1. Click **"Add Group"**
2. Fill in:
   - **Name** - Display name
   - **ID** - Unique identifier
   - **Order** - Position (lower = first)
   - **Icon** - Optional icon
3. Click **"Save"**

---

## üîß Managing Groups

### Reordering
- Drag groups to reorder
- Or edit the Order number
- Changes appear on Dashboard

### Editing
1. Find the group
2. Click **Edit**
3. Modify name or order
4. Save changes

### Deleting
1. Ensure no modules are in the group
2. Click **Delete**
3. Confirm

> ‚ö†Ô∏è Cannot delete groups with modules assigned.

## üì¶ Assigning Modules to Groups

### In Module Settings
1. Go to **Modules** management
2. Edit a module
3. Select the group
4. Save

### Bulk Assignment
1. Select multiple modules
2. Use bulk actions
3. Choose group
4. Apply

## ‚ùì FAQ

### Q: Can I have modules in multiple groups?
**A:** No, each module belongs to one group.

### Q: What if I delete a group?
**A:** You must first move modules to another group.

### Q: Groups not showing on Dashboard?
**A:** Empty groups may be hidden. Add modules to see them.

---

## üí° Best Practices

1. **Logical grouping** - Related modules together
2. **Clear names** - Users should understand purpose
3. **Limit groups** - Too many = confusing
4. **Order wisely** - Most used groups first
