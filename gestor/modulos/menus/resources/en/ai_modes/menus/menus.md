Generate an HTML page (only the inner part of the <body>) containing a site NAVIGATION MENU. This page will be used as a TEMPLATE in which an engine dynamically renders a LIST of menu items (links to site pages) curated by the user.

MANDATORY ITEM REPETITION RULE:
Every HTML snippet that represents ONE menu item (e.g. an `<li>`, a navbar link, a footer column) MUST be wrapped between the loop markers below:

```
<!-- item < -->
...HTML of ONE menu item, containing the placeholders @[[item#variable_name]]@...
<!-- item > -->
```

At runtime, the engine repeats the content between `<!-- item < -->` and `<!-- item > -->` once for each curated menu item, replacing each placeholder `@[[item#variable_name]]@` with the real value of the matching page field. Variables outside this block are NOT repeated (they serve the menu container, logo, header, etc).

If the user wants to show a message when the menu has no items, wrap that snippet in the optional block:

```
<!-- no-item < -->
...HTML shown when the menu has no items...
<!-- no-item > -->
```

PLACEHOLDER FORMATTING RULE:
- Always use the format `[[item#variable_name]]` (item is literal/static; variable_name is dynamic).
- Use the version without `@` for your output — the internal pipeline converts it to `@[[item#variable_name]]@` automatically on save.

AVAILABLE MENU ITEM VARIABLES:
- `[[item#label]]` — page label/name (link text)
- `[[item#url]]` — page address (use in the link `href` attribute)
- `[[item#target]]` — link target (`_self` or `_blank`); use in the link `target` attribute: `target="[[item#target]]"`
- `[[item#slug]]` — page textual identifier (optional)
- `[[item#css_classes]]` — extra CSS classes of the item (append to the link/element classes)
- `[[item#children]]` — recursive render of the sub-items (submenu); use ONLY inside the `item-parent` block

NESTED SUBMENUS (item with children):
For items that have a submenu, besides the `item` (leaf) block also define the `item-parent` block. Inside it, the `[[item#children]]` placeholder receives the recursive render of the children:

```
<!-- item-parent < -->
<li>
    <a href="[[item#url]]" target="[[item#target]]" class="[[item#css_classes]]">[[item#label]]</a>
    <ul>[[item#children]]</ul>
</li>
<!-- item-parent > -->
```

MENU DIVIDER/SEPARATOR (optional):
To render a visual divider between items, define the `item-separator` block. It can optionally show a label via `[[item#label]]` (which may be empty — in that case render only the physical line):

```
<!-- item-separator < -->
<li><span>[[item#label]]</span><hr></li>
<!-- item-separator > -->
```
