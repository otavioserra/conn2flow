#select-module#
Fields Schema
Data Information
JSON of All Parameters
Parameters Documentation
Basic Properties
Used to identify the form.
name
Descriptive form name
id
Unique form identifier
description
Optional form description
Name and Email Fields (Required)
Identifier of name and email fields in "fields" to be used in email communication.
field_name
Identifier of the name field in "fields"
field_email
Identifier of the email field in "fields"
Access Control (Optional)
Access controls are done by default, but can be configured as needed per form.
access_max_simple
Maximum simple submissions per IP (without CAPTCHA)
access_max
Maximum total submissions per IP before 24-hour block
force_recaptcha
Force use of reCAPTCHA v3 always
Email Configuration (Optional)
Email configuration is already done automatically in the Environment Config / Email Settings module. But it can be customized per form.
email.recipients
Recipients separated by ";"
email.reply_to
Reply-to email
email.subject
Email subject
email.template
ID of email template component
Redirects
Redirect URLs when there is success in a form. If there is failure, return to the same form. Or to the site root.
redirects.success.type
Type: "url" access that will be used for success redirect
redirects.success.path
Path for success redirect
redirects.error.type
Type: "url" access that will be used for error redirect
redirects.error.path
Path for error redirect
Form Fields
You can include as many fields as you want in each form, but must follow the standard configuration below.
Field Properties:
name
Field name (unique identifier)
label
Label displayed to the user
type
Field type (text, email, textarea.)
required
Required field
Library Functions
Sequence of execution of functions in the forms library to understand how each configuration above works.
formulario_controlador():
Prepares and configures forms on frontend
Loads HTML and JavaScript components
Configures client-side validations
Manages access permissions
Integrates reCAPTCHA v2/v3
Processes field schema and options
formulario_processador():
Processes form submissions on backend
Validates data and CAPTCHA
Applies rate limiting and anti-spam
Saves data in forms_submissions
Sends notification emails
Manages post-submission redirects
